# Licensing Strategy

This starter kit uses MIT by default.

## What MIT Allows

MIT allows others to:

- use the project
- copy it
- modify it
- redistribute it
- use it commercially
- sell derivative work

They must keep the copyright and license notice.

## What MIT Does Not Protect

MIT does not stop someone from building a similar project.
It does not give you control over derivatives after release.

If someone receives a MIT-licensed version, you generally cannot revoke that permission for that version later.

## Practical Strategy

Publish only what you are comfortable giving away.

Good to publish:

- starter vault
- templates
- essential skills
- adapters
- sample project
- docs

Keep private if you want future leverage:

- real project vaults
- client/team data
- advanced dashboards
- risky automation scripts
- private migration tools
- business-specific playbooks

## How To Still Make Money

Open source the starter.
Sell or offer:

- private setup
- team customization
- migration from existing AI chats
- Obsidian dashboard or plugin
- support
- workshops
- company policy packs

## Alternative Licenses

- MIT: easiest for adoption.
- Apache-2.0: more explicit patent language.
- Source-available / non-commercial: more control, less adoption.
- Dual license: possible, but more confusing for early users.

For early public adoption, MIT is usually the simplest choice.
