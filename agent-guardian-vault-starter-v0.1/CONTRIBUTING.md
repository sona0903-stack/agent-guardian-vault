# Contributing

Contributions are welcome.

## Principles

- Keep the starter small.
- Do not add runtime dependencies unless necessary.
- Prefer Markdown templates over executable code.
- Do not include real secrets, personal logs, or private project data.
- Keep agent-specific instructions in `agent_adapters/`.
- Keep reusable operating rules in `starter_vault/skills/`.

## Pull Request Checklist

- The change is useful for more than one agent or workflow.
- No real personal data is included.
- No secrets, tokens, or credentials are included.
- New graph links point to existing files.
- New docs explain why the file exists.
- README or docs are updated if behavior changes.
