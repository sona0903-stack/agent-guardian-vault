# SKILL_POLICY

Do not load every skill all the time.

## Always-On Skills

- `skills/essential_guardian.md`
- `skills/essential_coding_micro_harness.md`

## Optional Skills

Create optional skills only when the project needs them.

Examples:

- Android build
- Web frontend
- Python automation
- Data analysis
- Local webapp persistence
- Obsidian graph cleanup
- OpenHands compatibility
- Claude Code adapter
- Codex adapter
- Local LLM adapter

## Rule

Essential skills protect the workflow.
Optional skills add domain knowledge.

If a skill is not relevant to the active task, do not load it.
