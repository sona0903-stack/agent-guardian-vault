# CHANGE_GOVERNANCE

Change governance keeps the vault from becoming a pile of copied policies.

## Before Adding A Policy

Check whether an existing policy already has the same role.

If it does, update the existing policy and append a dated change note.
If it does not, create a new policy with a clear role-specific name.

## Conflict Table

Use this table when a design changes:

| --- | --- | --- |

## Required Change Record

Every meaningful policy change should record:

- date
- user request
- files changed
- reason
- compatibility concern
- proof or review status
